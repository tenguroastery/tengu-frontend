## 8. Setup de medición — GA4 + GTM + Meta CAPI

### GA4 eventos

```js
// view_item (ficha de producto)
gtag('event', 'view_item', {
  currency: 'CLP', value: 16500,
  items: [{
    item_id: 'TENGU-MG-RW-NAT-250',
    item_name: 'Marie Gorette Rwanda Natural',
    item_brand: 'Tengu Roastery',
    item_category: 'Filtrado',
    item_category2: 'Rwanda',
    item_variant: '250g',
    price: 16500, quantity: 1
  }]
});

// purchase
gtag('event', 'purchase', {
  transaction_id: 'TENGU-2026-001234',
  currency: 'CLP', value: 33000,
  shipping: 3500, tax: 0,
  items: [/* ... */]
});

// otros: add_to_cart, begin_checkout, newsletter_signup, whatsapp_click
```

> Ya implementado en `frontend/src/lib/analytics.ts` con consent gating del banner de cookies.

### GTM — variables y triggers

- **Variables**: `dataLayer.value`, `dataLayer.items`, `dataLayer.transaction_id`, `page_path`, `click_url`
- **Triggers**: Form Submit (newsletter), Click - Just Links (botón WhatsApp `wa.me`), Custom Event (purchase desde checkout), Page View (rutas específicas)
- **Tags**: GA4 Configuration + GA4 Event (uno por evento) + Meta Pixel + CAPI server-side via Stape o GTM Server

### Conversiones primarias en Google Ads

| Evento | Tipo | Valor |
|--------|------|-------|
| `purchase` | Primaria | dinámico |
| `begin_checkout` | Secundaria | – |
| `add_to_cart` | Secundaria | – |
| `newsletter_signup` | Secundaria | CLP 1.500 (proxy) |
| `whatsapp_click` | Secundaria | CLP 1.000 (proxy) |

### Meta CAPI (server-side)

Implementar Conversion API vía GTM Server o backend directo. Eventos: Purchase, InitiateCheckout, AddToCart, ViewContent, Lead. Recupera 10-25% de conversiones perdidas por iOS 14+ / bloqueadores.
